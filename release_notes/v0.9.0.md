# v0.9.0

## Breaking change: `FlowBuilder.use()` removed — use `FlowBuilder.extend()` instead

The global `FlowBuilder.use(plugin)` method has been removed. It mutated the
`FlowBuilder` prototype globally, meaning any plugin registered in one part of
an application would affect all flows everywhere — a source of subtle bugs,
test pollution, and ordering issues.

The replacement is `FlowBuilder.extend([...plugins])`, which has been available
since v0.6.0. It returns a **new subclass** with the plugins mixed into its
prototype. The base `FlowBuilder` class is never touched, so every
`extend()` call is fully isolated.

### Migration

**Before (v0.8 and earlier)**

```typescript
import { FlowBuilder } from "flowneer";
import { withRateLimit } from "flowneer/plugins/llm";
import { withTiming } from "flowneer/plugins/observability";

FlowBuilder.use(withRateLimit);
FlowBuilder.use(withTiming);

const flow = new FlowBuilder<MyState>()
  .then(step)
  .withRateLimit({ intervalMs: 500 })
  .withTiming();
```

**After (v0.9+)**

```typescript
import { FlowBuilder } from "flowneer";
import { withRateLimit } from "flowneer/plugins/llm";
import { withTiming } from "flowneer/plugins/observability";

const AppFlow = FlowBuilder.extend([withRateLimit, withTiming]);

const flow = new AppFlow<MyState>()
  .then(step)
  .withRateLimit({ intervalMs: 500 })
  .withTiming();
```

### Why `extend()` is better

|               | `use()` (removed)                    | `extend()`                                     |
| ------------- | ------------------------------------ | ---------------------------------------------- |
| Scope         | Global — affects all flows           | Scoped to the returned subclass                |
| Isolation     | None — test pollution, ordering bugs | Full — each `extend()` call is independent     |
| Composability | Flat list, last-write-wins           | Chainable subclasses (`AppFlow.extend([...])`) |
| Tree-shaking  | Plugins always included              | Only included when subclass is used            |

### Chaining subclasses

`extend()` is polymorphic — call it on a subclass to layer more plugins on top:

```typescript
const BaseFlow = FlowBuilder.extend([withRateLimit, withTiming]);
const TracedFlow = BaseFlow.extend([withTrace]); // has all three plugins
```

### `withPlugin` / instance `.with()` also removed

Any instance-level `.with(plugin)` or `withPlugin()` patterns that were
available in earlier pre-release versions are also gone. `extend()` is the
single, canonical way to attach plugins to a flow class.
